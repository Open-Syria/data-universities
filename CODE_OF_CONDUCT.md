# Code of Conduct

OpenSyria Data Universities is built for public benefit. We want contribution spaces to be respectful, careful, and useful for people contributing from different backgrounds, languages, locations, and levels of technical experience.

## Table of Contents

- [Expected Behavior](#expected-behavior)
- [Unacceptable Behavior](#unacceptable-behavior)
- [Dataset Safety](#dataset-safety)
- [Enforcement](#enforcement)
- [Scope](#scope)

## Expected Behavior

- Be respectful and constructive.
- Discuss data, sources, licensing, and review decisions without personal attacks.
- Keep feedback specific and actionable.
- Respect project scope, source licensing, and safety boundaries.
- Accept that maintainers may reject data that is unsafe, unsourced, restricted, or outside the approved schema.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private information without permission.
- Sharing secrets, credentials, private infrastructure details, or personal data.
- Pressuring maintainers to accept out-of-scope or unsafe data.
- Repeatedly ignoring maintainer decisions or repository contribution rules.

## Dataset Safety

Do not contribute, request, or discuss publication of:

- personal data,
- private addresses, phone numbers, IDs, account data, or student records,
- security, military, checkpoint, surveillance, or operational-sensitive data,
- restricted or proprietary data,
- unsourced political claims.

## Enforcement

Maintainers may edit, hide, lock, or remove comments, issues, discussions, or pull requests that violate this code of conduct. Maintainers may also block users from participating in the repository when needed.

Security issues should be reported privately through [SECURITY.md](SECURITY.md), not through public issues.

## Scope

This code of conduct applies to this repository and related OpenSyria project spaces.
